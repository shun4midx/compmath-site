Example 5.6.1

The antiderivative of $e^x$ is, of course, itself. That makes evaluation of $_0^1 e^x dx$ the Fundamental Theorem trivial.

In[]:
exact = np.exp(1) - 1
print(exact)

1.718281828459045

The module scipy.integrate has multiple functions that estimate the value of an integral numerically without finding the antiderivative first. As you can see here, its often just as accurate.

In[]:
from scipy.integrate import quad
Q, errest = quad(np.exp, 0, 1, epsabs=1e-13, epsrel=1e-13)
print(Q,errest)

1.7182818284590453 1.9076760487502457e-14

The numerical approach is also far more robust. For example, $e^{sinx}$ has no useful antiderivative. But numerically, its no more difficult.

In[]:

print(Q)

1.6318696084180513

When you look at the graphs of these functions, whats remarkable is that one of these areas is basic calculus while the other is almost impenetrable analytically. From a numerical standpoint, they are practically the same problem.

Trapezoid formula

In[]:
def trapezoid(f, a, b, n):
    """
    trapezoid(f, a, b, n)

    Apply the trapezoid integration formula for integrand f over interval [a,b], broken up into n equal pieces. Returns estimate, vector of nodes, and vector of integrand values at the nodes.
    """
    h = (b - a) / n
    t = np.linspace(a, b, n + 1)
    y = f(t)
    T = h * (np.sum(y[1:-1]) + 0.5 * (y[0] + y[-1]))
    return T, t, y

We will approximate the integral of the function $f(x)=e^{sin(7x)}$ over the interval $[0,2]$.

In[]:
f = lambda x: np.exp(np.sin(7 * x))
a, b = 0, 2

In lieu of the exact value, we will use the quad function to find an accurate result.

Here is the trapezoid result at n=40, and its error.

In[]:

print(f"Trapezoid estimate is {T:.14f} with error {I - T:.2e}")

Trapezoid estimate is 2.66230293560229 with error 9.17e-04

In order to check the order of accuracy, we increase n by orders of magnitude and observe how the error decreases.

In[]:
n_ = 40 * 2 ** np.arange(6)
err = np.zeros(np.size(n_))
print("     n     error")
for k, n in enumerate(n_):
    T, t, y = trapezoid(f, a, b, n)
    err[k] = I - T
    print(f"{n:6d}   {err[k]:8.3e} ")

Each increase by a factor of 10 in n cuts the error by a factor of about 100, which is consistent with second-order convergence. Another check is that a log-log graph should give a line of slope -2 as n.

Example 5.6.3

We estimate $_0^2 x^2 e^{-2x} dx$ using extrapolation. First we use quadgk to get an accurate value.

In[]:
from scipy.integrate import quad
f = lambda x: x**2 * np.exp(-2 * x)
a = 0
b = 2
I, errest = quad(f, a, b, epsabs=1e-13, epsrel=1e-13)
print(f"Integral = {I:.14f}")

Integral = 0.19047417361161

We start with the trapezoid formula on $n=N$ nodes.

In[]:
N = 20    # the coarsest formula
n = N
h = (b - a) / n
t = h * np.arange(n + 1)
y = f(t)

We can now apply weights to get the estimate $T_f(N)$.

In[]:
T = np.zeros(3)
T[0] = h * (sum(y[1:-1]) + y[0] / 2 + y[-1] / 2)
print(f"error (2nd order): {I - T[0]:.2e}")

error (2nd order): 6.27e-05

Now we double to $n=2N$, but we only need to evaluate $f$ at every other interior nod

In[]:
n = 2 * n
h = h / 2
t = h * np.arange(n + 1)
T[1] = T[0] / 2 + h * sum(f(t[1:-1:2]))
print("error (2nd order):", I - T[:2])

error (2nd order): [6.27236723e-05 2.75002671e-05]

As expected for a second-order estimate, the error went down by a factor of about 4. We can repeat the same code to double $n$ again.

In[]:
n = 2 * n
h = h / 2
t = h * np.arange(n + 1)
T[2] = T[1] / 2 + h * sum(f(t[1:-1:2]))
print("error (2nd order):", I - T[:3])

error (2nd order): [6.27236723e-05 2.75002671e-05 1.27933290e-05]

Let us now do the first level of extrapolation to get results from Simpson's formula. We combine the elements T[i] and T[i+1] the same way for i=1 and i=2.

In[]:
S = np.array([(4 * T[i + 1] - T[i]) / 3 for i in range(2)])
print("error (4th order):", I - S)

error (4th order): [1.57591320e-05 7.89101624e-06]

With the two Simpson values $S_f(N)$ and $S_f(2N)$ in hand, we can do one more level of extrapolation to get a sixth-order accurate result.

In[]:
R = (16 * S[1] - S[0]) / 15
print("error (6th order):", I - R)

error (6th order): 7.366475186759702e-06

import numpy as np
import matplotlib.pyplot as plt
from scipy import linalg as LA
from time import perf_counter as timer
N=np.arange(1,501,50)
rng = np.random.default_rng()
t1 = []
t2 = []
for n in N:
  A = rng.random((n,n))
  b = np.ones((n,1))
  tic = timer()
  x1 = LA.solve(A, b)
  toc = timer()
  t1.append(toc-tic)
  tic = timer()
  x2 = LA.inv(A)*b
  toc = timer()
  t2.append(toc-tic)
plt.plot(N,t1, '-ro',N,t2, '-bx')
plt.xlabel('Matrix dimension')
plt.ylabel('Time (s)')
plt.legend(['linalg.solve(A, b)',
'linalg.inv(A)*b'])
plt.grid('on')
plt.show()

import numpy as np
import matplotlib.pyplot as plt
from matplotlib.widgets import Slider
from PIL import Image
from scipy.linalg import svd

import requests
from io import BytesIO

#A = Image.open('flowers.jpeg').convert('L')

url = 'https://www.math.ntu.edu.tw/sites/default/files/math_logo_w150_8081.jpg'
response = requests.get(url)
img = Image.open(BytesIO(response.content))
A = img.convert('L')

U, s, VT = svd(A)
def resize_image(N):
  return U[:,:N] @ np.diag(s[:N]) @ VT[:N,:]
fig,(ax1, ax2) = plt.subplots(1,2,
  figsize=(10,4))
plt.subplots_adjust(bottom=0.2)
ax1.set_xlim(1,len(s))
ax1.title.set_text('Singular values')
ax1.grid('on')
ax1.loglog(s,'ro-', markersize=3)
N = 10
image = resize_image(N)
ax2.imshow(image, cmap='gray')
axt = plt.axes([0.125, 0.1, 0.355, 0.05])
N_slide = Slider(axt, 'N',
  0, np.log10(len(s)),
  valstep=0.05, valinit=1)
N_slide.valtext.set_text(N)
def update(val):
  N = round(10**N_slide.val)
  N_slide.valtext.set_text(N)
  image = resize_image(N)
  ax2.imshow(image, cmap='gray')
  fig.canvas.draw_idle()
N_slide.on_changed(update)
plt.show()
