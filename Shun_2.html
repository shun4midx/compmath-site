# GramSchmidt orthogonalization
def gram_schmidt(vectors):
  """Orthonormalize a set of vectors using the Gram-Schmidt process."""
  vectors = [np.array(v, dtype=float) for v in vectors]
  ortho = []
  for v in vectors:
    w = v - sum(np.dot(v, u)*u for u in ortho)
    if norm(w) > 1e-10:
      ortho.append(w / norm(w))
  return np.array(ortho)

def F(n):
    if (n == 1):
        return 1

    prev = 0
    curr = 1
    
    while (n > 1):
        next = curr + prev
        prev = curr
        curr = next
        n -= 1
    
    return next

Case: finding the roots of a quadratic polynomial

We apply the quadratic formula to find the roots of a quadratic $$p(x) = (x-10^{6})(x-10^{-6})$$

In[6]:
a = 1;  b = -(1e6 + 1e-6);  c = 1;
x1 = (-b + np.sqrt(b**2 - 4*a*c)) / 2*a
x2 = (-b - np.sqrt(b**2 - 4*a*c)) / 2*a
print(x1)
print(x2)

1000000.0
1.00000761449337e-06

The first value is correct to all stored digits, but the second has fewer than six accurate digits:

In[7]:
error = abs(1e-6 - x2) / 1e-6
print(f"There are {-np.log10(error):.2f} accurate digits.")

There are 5.12 accurate digits.

The instability is easily explained. Since a=c=1, we treat them as exact numbers. First, we compute the condition numbers with respect to b for each elementary step in finding the good root: \begin{array}{|l|l|l|} \hline \textbf{Calculation} & \textbf{Result} & \boldsymbol{\kappa} \\ \hline u_1 = b^2 & 1.000000000002000 \times 10^{12} & 2 \\ u_2 = u_1 - 4 & 9.999999999980000 \times 10^{11} & \approx 1.00 \\ u_3 = \sqrt{u_2} & 999999.9999990000 & 1/2 \\ u_4 = u_3 - b & 2000000 & \approx 0.500 \\ u_5 = u_4 / 2 & 1000000 & 1 \\ \hline \end{array}

Using (1.2.9), the chain rule for condition numbers, the conditioning of the entire chain is the product of the individual steps, so there is essentially no growth of relative error here. However, if we use the quadratic formula for the bad root, the next-to-last step becomes $$ u_4 = (-u_3) - b, $$ and now $$ \kappa = \frac{|u_3|}{|u_4|} \approx 5 \times 10^{11}. $$ So we can expect to lose 11 digits of accuracy, which is what we observed. The key issue is the subtractive cancellation in this one step.

New Roots: [-1.00000000e+14-1286321.55392366j -1.00000000e+14+1286321.55392366j
  9.99999989e-15      +0.j          1.00000001e-14      +0.j        ]

As the chosen roots have very large magnitudes $(\approx 10^{14})$ and very small ones $(\approx 10^{-14})$, the resulting polynomial coefficients span many orders of magnitude. This makes the polynomial highly ill-conditioned. Additionally, adding repeated roots amplifies sensitivity: tiny perturbations in the coefficients, which are unavoidable in floating-point arithmetic, can cause the computed roots to shift significantly. Hence, the algorithm produces large forward errors in the roots, even though the backward error remains small.

Here, we created the example of a polynomial with high backward error, up to $1$. This is done via getting nearby roots differing by a small margin, including $-\varepsilon_{mach}$ and $\varepsilon_{mach}$, while also including large contrasting roots such as $1-10^8$ and $1 + 10^8$. Then, when multiplying the roots to form the polynomial, these form coefficients with order of magnitude $\varepsilon_{mach}$. As the terms will be negligible when computing the roots of the polynomial, it causes huge backward error.

def forwardsub(L,b):
  """
  forwardsub(L,b)

  Solve the lower-triangular linear system with matrix L and right-hand side
  vector b.
  """
  n = len(b)
  x = np.zeros(n)
  for i in range(n):
    s = L[i,:i] @ x[:i]
    x[i] = ( b[i] - s ) / L[i, i]
  return x
def backsub(U,b):
  """
  backsub(U,b)

  Solve the upper-triangular linear system with matrix U and right-hand side
  vector b.
  """
  n = len(b)
  x = np.zeros(n)
  for i in range(n-1, -1, -1):
    s = U[i, i+1:] @ x[i+1:]
    x[i] = ( b[i] - s ) / U[i, i]
  return x
